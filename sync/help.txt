<tr><th style="text-align:left">Введение</th></tr>
<tr><td style="text-align:left">
Система синхронизации (СС) предназначена для импорта данных в СКУД о подразделениях, сотрудниках и отпусках из
кадровых систем. Обмен происходит с помощью файлов определённого формата (см. описание протокола файлов).<br>
Синхронизация выполняется по расписанию, которое задаётся администратором СКУД.
</td></tr>

Для выгрузки наработки необходимо выбрать любую дату нужного месяца.

Позволяет просматривать логи системы синхронизации. Для просмотра лога укажите период, который Вы хотите просмотреть.

<tr><th style="text-align:left">Вкладка &quot;Настройки&quot;</th></tr>
<tr><td style="text-align:left">
На вкладке отображается настройки СС.
<br>Настройки:
<ol>
<li><b>Путь к файлу сотрудников</b> - полный путь к файлу (вместе с именем файла), содержащим информацию о сотрудниках. Путь должен указывать на файл, который находится на локальной машине! </li>
<li><b>Путь к файлу подразделений</b> - полный путь к файлу (вместе с именем файла), содержащим информацию о подразделениях. Путь должен указывать на файл, который находится на локальной машине! </li>
<li><b>Тип разделителей</b> - указывает на то, какие будут разделители строк и колонок в файлах.</li>
<li><b>Разделитель строк</b> - символ или код символа который является разделителем строк. Если в качестве разделителя выбираете код символа, то значением параметра "Тип разделителей" должно быть "код символа"</li>
<li><b>Разделитель колонок</b> - символ или код символа, который является разделителем колонок. Если в качестве разделителя выбираете код символа, то значением параметра "Тип разделителей" должно быть "код символа" </li>
<li><b>Преобразование кодовой страницы</b> - для нормального (а не абракадабра) отображения данных,  импортированных из файлов, оставьте это значение равным "ACP-windows 1251"</li>
<li><b>Удаление данных</b> - если галочка установлена, то при импорте данных будут удалены данные, которые отсутствуют в файлах (кроме тех, которые были заведены через СКУД). Удаление данных позволяет предотвратить разбухание таблиц БД. </li>
<li><b>Очищать лог при каждом запуске</b> - если галочка установлена, то лог будет очищаться при каждом запуске синхронизации. </li>
<li><b>Очистить таблицы перед импортом</b> - если галочка установлена, данные о сотрудниках и подразделениях будут удалены.</li>
<li><b>Путь к файлу наработки</b> - полный путь к папке с файлом (без имени файла), содержащим информацию о наработке сотрудников. Путь должен указывать на папку, которая находится на локальной машине! </li>
</ol>
</td></tr>
